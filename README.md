# nix-kanata

A `nix-darwin` module to configure and manage [kanata](https://github.com/jtroo/kanata) and [kanata-tray](https://github.com/rszyma/kanata-tray) on macOS.

This module handles Homebrew dependencies (`karabiner-elements`, `kanata-tray`), symlinks your configurations, and generates macOS-friendly transparent icons for the menu bar on the fly.

## Prerequisites

Kanata requires `karabiner-elements` to intercept keys on macOS. This module installs it via Homebrew, so you must have `homebrew.enable = true;` in your `nix-darwin` configuration.

## Installation

Add the flake to your inputs and include the module in your `darwinConfigurations`:

```nix
{
  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixpkgs-unstable";
    nix-darwin.url = "github:LnL7/nix-darwin";
    nix-kanata.url = "github:iluvshiwoon/nix-kanata";
  };

  outputs = { self, nix-darwin, nixpkgs, nix-kanata, ... }: {
    darwinConfigurations."macbook" = nix-darwin.lib.darwinSystem {
      system = "aarch64-darwin"; # or "x86_64-darwin"
      modules = [
        ./configuration.nix
        nix-kanata.darwinModules.default
      ];
    };
  };
}

```

### Important: macOS Permissions & First Run

Because Kanata relies on Karabiner-Elements' virtual keyboard drivers, the first deployment requires manual intervention:

1. **Sudo Prompts:** During `darwin-rebuild switch`, you will be prompted for your `sudo` password multiple times. Homebrew requires this to install the Karabiner system drivers.
2. **Approve Extensions:** Go to **System Settings > General > Login Items & Extensions** and ensure Karabiner-related background items are allowed.
3. **Input Monitoring:** Open the **Karabiner-Elements** app manually once. Grant "Input Monitoring" permissions in **System Settings > Privacy & Security > Input Monitoring** for both Karabiner and Kanata.
4. **Restart Kanata-Tray:** If the icons don't appear or keys aren't remapping, quit `kanata-tray` from the menu bar or run `pkill -x kanata-tray`. macOS `launchd` will restart it automatically with the new permissions.

## Usage

Example configuration using **Fira Code Nerd Font** for "liquid glass" style (transparent) icons:

```nix
{ config, pkgs, ... }:

{
  # Install the font package
  fonts.packages = [ pkgs.nerd-fonts.fira-code ];

  services.kanata = {
    enable = true;
    
    # Use pkgs.kanata-with-cmd for shell command support
    enableCmd = true; 
    
    # "tray" for menu bar app, "daemon" for headless root service
    mode = "tray"; 

    # Path to your .kbd file
    configSource = ./kanata.kbd; 

    tray = {
      autostart = true;

      icons = {
        # Absolute path to the .ttf or .otf file
        font = "${pkgs.nerd-fonts.fira-code}/share/fonts/truetype/NerdFonts/FiraCode/FiraCodeNerdFont-Bold.ttf";

        # Map layer names to text or Unicode hex codes (e.g., Nerd Font icons)
        labels = {
          qwerty  = "QWE";
          nav     = "U+F0B2"; # ¾░é▓
          sym     = "U+F121"; # ’äĪ
        };

        # Strict status keys: "default", "pause", "crash", "live-reload"
        status = {
          default       = "U+F11C"; # ’ä£ 
          pause         = "U+F04C"; # ’üī
          crash         = "U+F071"; # ’ü▒
          "live-reload" = "U+F021"; # ’ĆĪ
        };
      };
    };
  };
}

```

### Finding the Exact Font Path

To find the internal path of a font in the Nix store, run:

```bash
find $(nix-build --no-out-link -E '(import <nixpkgs> {}).nerd-fonts.fira-code') -name "*.ttf"

```

## Options Reference

* `services.kanata.enableCmd`: If `true`, adds `pkgs.kanata-with-cmd` to system packages.
* `services.kanata.mode`: `"tray"` (default) or `"daemon"`.
* `services.kanata.tray.icons.font`: Path to font file (Required for icon generation).
* `services.kanata.tray.icons.status`: Map for status icons. Keys must be `default`, `crash`, `pause`, or `live-reload`.

## Implementation Notes

* **Icon Generation:** If icon labels or statuses are defined, the module uses `imagemagick` during evaluation to render 128x128 PNGs with a transparent background.
* **Fallback Cleanup:** During the activation script, the module deletes any default `.ico` files generated by `kanata-tray` in the `status_icons` directory so the Nix-generated `.png` files take priority.
* **Reloading:** Applying a new configuration (`darwin-rebuild switch`) automatically kills the running `kanata-tray` process at the end of the activation script. `launchd` immediately restarts it, picking up the new `kanata.kbd` and icons.

## Uninstallation (The Nix Way)

This module respects the declarative "Nix way". Setting `services.kanata.enable = false;` unloads all `launchd` services and removes symlinks. To completely remove the Homebrew drivers and apps, ensure your cleanup is set to `"zap"`:

```nix
homebrew.onActivation.cleanup = "zap";

```
